2018年,我的开发状况

2018年，我的变化挺大的，技术栈从Python慢慢转向Python+JavaScript，过去开发，后端用Python，然后基于Python的模板引擎渲染html页面. 现在，前端使用react框架和antd组建库，用fetch去跨域请求，后端或用sanic或用flask或用nodejs。

后端接受跨域请求的方法很简单:在response的headers的'Access-Control-Allow-Origin'设置为'*', 这里的'*'表示所有路由，你也可以选择你要实现跨域的路由。

另一个大变化是，我开始使用docker了。

docker确实给我带来了很大的方便，我现在的数据库全部开在docker容器中，

这是我平常用的镜像.

我也写了个Makefile，用来记下镜像打开的方式。

➜ ltoddy@linuxbox  ~  cat Makefile
run:
	docker run --name mysql -d -p 3306:3306 -e MYSQL_ROOT_PASSWORD=hello  mysql:5.7
	docker run --name mongo -d -p 27017:27017 mongo:3.6
	docker run --name redis -d -p 6379:6379 redis:4.0.8
	docker run --name rabbitmq -d -p 4369:4369 abbitmq:3.7

倒不是我不知道怎么run镜像，而是写在这里，方便我复制粘贴，我的容器不是长时间开着的，我没有设置'--restart=always'字段， 我的容器基本上没一会就会被我销毁的，毕竟我目前还没做什么项目，仅是在练习。

docker容器化确实带来了很多好处,每个应用可以有自己的运行环境，也不会有什么端口被占用之类的问题。

总之docker的便利，你用了才会知道，反正我感觉是太好使了。

这个学期,并没有像过去那样天天写代码了，我现在很少写代码了，更多的是看理论知识了，也许这算是比以前更加成长，有了过去打下的基础， 能够更容易的掌握知识。
